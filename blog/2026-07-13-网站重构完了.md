# 重构史山网站有感

要不是零音在群里喊了一声「谁要友链」，岩还真想不起来自己有个站了。

脑袋里就一直想着要把 last generation 留下来的网站重构一下，一直想了又忘……
没招了，今天趁着有时间，想着要治治自己的拖延症，所以过来审视一下当时的史山。

这样便有了这篇帖子。

<!-- more -->

## 什么叫你手写的 blog sidebar？

看一个 VitePress 网站总是要从 `.vitepress` 看起的。

翻至 `.vitepress/config.mts`，眼前的场景令岩眼前一黑——

**「WTF，这一大坨是啥？」** 岩爆了粗口。

令岩如此震惊的便是 `config.themeConfig.sidebar` 中排山倒海的树状结构。
Last generation 他™的把所有的东西都列在了 sidebar 配置里面，导致配置文件加起来有惊人的 200+ 行。

岩让 Codex 把它改成了用 Teek 主题提供的 resolver 来动态生成 sidebar。

## 我艹这个阴影真的太神秘了

翻至 blog 主页，映入眼帘的便是一坨长了毛边的卡片——？

岩切到亮色模式，无事发生。岩又切回暗色模式，「毛边」出现了。

敏锐的直觉告诉岩，这是阴影没适配颜色模式导致的——

「雾雨，帮我把这个阴影调一下」岩打开 VSCode 的侧边栏，在 Codex 中发出指令。

强悍的 GPT-5.6-Sol 很快解决了问题。

***不过，岩是不是有点太依赖 Codex 了呢，好像有点变笨了🤔***

## 切个 Catppuccin 主题

岩一直很喜欢 Catppuccin 的配色！

岩去搜 VitePress 自定义主题的方法，发现直接在 `.vitepress/theme/index.mts` 内引入 CSS 就行了。

接着岩打开了 Catppuccin Palettes，然后就对着满屏的调色盘犯了难。

「还是交给 Codex 干吧」岩想，手又不自觉地挪到了输入框上——

敲了几次回车授权访问网络，Codex 写了一个 200 多行的 CSS，岩刷新页面，效果很好。

岩加了个字体，然后关掉了 CSS 编辑页面。

## 忙完了。

岩靠在椅子上，呼了口气。

这次大概真的是忙完了吧。

回头一看，好像也没做什么惊天动地的大事嘛，也就是重构了一下 last generation 的史山网站罢了——

「可是，这些真的是岩做的么？」

如今的 Agent 已经非常强大，岩只需要动动手指告诉 Codex 需求，所有的任务就会自行完成。
Agent 已经把原来从「发现问题」到「解决问题」之间，那段跌跌撞撞的路，压缩成了几轮对话。

这当然很爽。

可是，网站到底是谁重构的？

Sidebar 的 resolver 是 Codex 调的，阴影配色是 Codex 改的，主题色是 Codex 配的。
岩就像一个什么都不会、但是意见很多甲方，絮絮叨叨地说：

「这里改一下」

「那里也改一下」

「嗯，不错」

岩甚至很难说清楚这个网站的源码里哪里到底在做什么，每一个 CSS 变量到底覆盖的是哪里的配置文件——

岩感觉自己确实变笨了。Last generation 曾经只凭借着搜索引擎写出了大量的不算优雅但是工作的很好的代码，
换作现在的岩貌似已经很难做到了。

岩瘫在椅子上。

总之，网站活过来了，blog 重新铲除了一篇文章。那些不认识的代码，以后再慢慢认识吧。
Codex，就少用一点好了。

「还好，至少点子貌似都还是我想的。」岩感慨。
